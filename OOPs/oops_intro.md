Introduction to OOP
OOP (Object-Oriented Programming) is a programming paradigm that organizes code around objects rather than functions and logic.
4 Pillars:

Encapsulation – Bundling data + methods inside a class; hiding internal details
Abstraction – Exposing only what's necessary, hiding complexity
Inheritance – A class can derive properties/methods from another class
Polymorphism – Same interface, different behavior (method overriding/overloading)

Core Concepts:

Class → Blueprint / template
Object → Instance of a class
Constructor → Special method to initialize an object
this keyword → Refers to the current object instance

Why OOP?

Reusability – Write once, reuse via inheritance
Modularity – Code is organized in self-contained units
Scalability – Easy to add features without breaking existing code
Maintainability – Easier to debug, update, and collaborate on

Key Mindset:

Everything in the real world is an object with properties (what it has) and behaviors (what it does). OOP mirrors real life — that's why OOP is life.
